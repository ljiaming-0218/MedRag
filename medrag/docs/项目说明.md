# DocRAG Agent：面向 PDF 文献阅读的 RAG 智能助手

DocRAG Agent 是一个通用 PDF 文献阅读助手，用于辅助理解学术论文、课程资料、技术文档、项目报告和政策文件等文本型 PDF。系统通过检索增强生成（RAG）把用户问题与文档原文片段结合，生成回答并展示引用来源。

系统回答仅用于辅助阅读，重要结论应结合原始文档和引用片段进行核验。

## 当前阶段

项目已完成基础 RAG 闭环和可演示的多轮问答链路，并已通过 Docker 部署到 Hugging Face Space。

已实现功能：

1. PDF 上传、文本解析和页码保留。
2. 基于 `chunk_size` 和 `chunk_overlap` 的文本切分。
3. 根据 PDF 内容 SHA-256 生成稳定的 `document_id`。
4. BGE Embedding 生成与 Chroma 向量存储。
5. 向量召回、文档过滤和 CrossEncoder 重排序。
6. 基于检索片段、历史消息和当前问题的 Prompt 组装。
7. 通过 OpenAI-compatible API 调用大模型。
8. MongoDB 持久化 conversation、user/assistant message 和 assistant sources。
9. 按 `conversation_id` 读取最近历史并进行多轮问答。
10. FastAPI 托管前端页面，支持 PDF 索引、检索调试、问答和引用展示。

## RAG 主链路

```text
上传 PDF
→ 计算 document_id 并保存文件
→ 按页解析文本
→ 切分为 chunks
→ 生成 Embedding
→ 写入 Chroma
→ 根据问题进行向量召回
→ CrossEncoder Rerank
→ 组装历史、文献片段和当前问题
→ 调用 LLM
→ 保存回答与 sources
→ 返回结构化结果
```

## 核心数据

### conversations

- `_id`：会话唯一标识。
- `document_id`：会话绑定的 PDF 文档。
- `title`：会话标题。
- `created_at`：创建时间。
- `updated_at`：最后更新时间。

### messages

- `_id`：消息唯一标识。
- `conversation_id`：所属会话。
- `role`：`user` 或 `assistant`。
- `content`：消息内容。
- `sources`：assistant 回答对应的文献片段。
- `created_at`：创建时间。

## 核心接口

| 接口 | 功能 |
| --- | --- |
| `GET /health` | 检查后端服务状态 |
| `POST /pdf/parse` | 解析 PDF 并返回分页文本 |
| `POST /pdf/chunks` | 查看文本切分结果 |
| `POST /pdf/index` | 解析、切分、向量化并建立索引 |
| `POST /pdf/search` | 检索相关文献片段 |
| `POST /pdf/prompt-preview` | 预览检索结果和 RAG Prompt |
| `POST /pdf/answer` | 执行单轮 RAG 问答 |
| `POST /conversations` | 创建文档会话 |
| `GET /conversations` | 获取会话列表 |
| `GET /conversations/{conversation_id}/messages` | 获取会话历史消息 |
| `POST /conversations/{conversation_id}/ask` | 在指定会话中继续提问 |

## 工程边界与风险

1. `.env` 不能提交真实 API Key，前端不得读取或保存 API Key。
2. 上传 PDF、Chroma 索引和模型缓存属于运行时数据，不应提交到 Git。
3. Hugging Face 免费实例的本地文件不能视为可靠持久化存储。
4. 当前主要支持文本型 PDF，扫描版 PDF 尚未接入 OCR。
5. 当前会话查询未按 `user_id` 隔离，不适合存储敏感或多用户数据。
6. 多轮问答会读取最近历史，但当前问题尚未经过 Query Rewrite。
7. 尚未建立系统化 RAG 评估集和完整自动化测试。

## 后续开发顺序

1. 建立轻量 `users` 数据结构和 `user_id`。
2. 为 document、conversation 和 message 增加用户归属与查询隔离。
3. 增加 `document_hash` 和同一用户下的文档去重识别。
4. 在前端展示当前用户的历史文档、会话和消息。
5. 加入 Query Rewrite，将多轮追问改写为适合检索的独立问题。
6. 根据 `user_type` 调整回答风格和内容侧重。
7. 实现轻量 Agent Router 和文献阅读报告工具。
8. 建立小型 RAG 评估集并增加自动化测试。
