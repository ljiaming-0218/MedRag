# MedRAG 医学文献知识库问答系统

本项目面向医学论文阅读场景，目标是构建一个基于 RAG 的医学文献辅助阅读系统。系统用于辅助理解文献，不替代医生判断，也不输出临床诊断或治疗建议。

## 当前阶段判断

当前代码已经超过“PDF 解析”阶段，进入了基础 RAG 闭环阶段：

1. PDF 文件上传
2. PDF 文本解析
3. 文本切分
4. embedding 生成
5. Chroma 向量化存储
6. 向量检索
7. rerank
8. prompt 组装
9. 大模型回答
10. 前端单轮问答演示

## 核心接口

1. `/pdf/parse`：解析 PDF 文本。
2. `/pdf/chunks`：查看切分结果。
3. `/pdf/index`：建立向量索引。
4. `/pdf/search`：检索相关文献片段。
5. `/pdf/prompt-preview`：预览 prompt。
6. `/pdf/answer`：生成文献问答结果。

## 工程风险

1. `backend/.env` 中不能提交真实 API key。
2. `backend/data/` 和 Chroma 数据属于运行时数据，不应提交。
3. `document_id` 当前由 PDF 内容哈希生成，适合避免同一文件重复生成不同 id。
4. 重建同一文档索引时会先删除同一 `document_id` 的旧 chunk，避免重复数据。
5. embedding 和 rerank 模型依赖本地缓存，换机器后需要重新确认模型可用性。

## 下一步

优先补最小接口测试，验证 `/pdf/index`、`/pdf/search`、`/pdf/answer` 能稳定跑通，再继续优化检索质量和医学安全提示。
