# 系统架构图

## 当前 RAG 流程

```text
用户上传 PDF
   ↓
FastAPI 接收文件
   ↓
保存 PDF，并用文件内容哈希生成 document_id
   ↓
PyMuPDF 提取每页文本
   ↓
按页切分 chunk
   ↓
sentence-transformers 生成 embedding
   ↓
写入 Chroma 持久化向量库
   ↓
用户输入问题
   ↓
问题向量化
   ↓
Chroma 按 document_id 检索候选 chunk
   ↓
CrossEncoder rerank
   ↓
组装医学问答 prompt
   ↓
OpenRouter/OpenAI-compatible API 生成回答
   ↓
返回答案和引用片段
```

## 主要模块边界

```text
routers/pdf_router.py
   只负责 HTTP 参数接收、调用服务、组织响应

services/pdf_upload.py
   校验并保存 PDF，生成 document_id

services/pdf_service.py
   解析 PDF 页面文本

services/text_splitter.py
   切分文本 chunk

services/embedding_service.py
   生成文本向量

services/vector_store_service.py
   读写 Chroma 向量库

services/search_service.py
   组合 query embedding、向量检索、过滤和 rerank

services/prompt_service.py
   组装 RAG prompt

services/llm_service.py
   调用大模型接口
```
